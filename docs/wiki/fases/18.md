## O que foi implementado

- Módulo `src/docagent/audio/` completo (models, schemas, services, router, stt/, tts/)
- 5 endpoints REST para gerenciar `AudioConfig` por tenant e por agente
- STT: FasterWhisper (singleton, thread pool) + OpenAI Whisper API
- TTS: Piper (subprocess + ffmpeg WAV→OGG) + OpenAI TTS + ElevenLabs
- Cascata de config: agente → tenant → system defaults (SimpleNamespace)
- WhatsApp: detecção `audioMessage`, download base64, transcrição, resposta
- Telegram: detecção `voice`/`audio`, getFile + download CDN, transcrição, resposta
- Frontend: `AudioConfigForm.vue`, aba "Áudio" em Settings, card em AgenteFormView
- Migração Alembic: `k1l2m3n4o5p6_add_audio_config.py`
- Testes: 7 arquivos em `tests/test_audio/`

## Bugs encontrados e corrigidos

1. FK `agentes.id` → `agente.id` (tablename singular) — models.py + alembic migration
2. `AudioConfig.__new__()` → `types.SimpleNamespace` para system defaults
3. `Tenant(slug=...)` → `Tenant(nome=...)` — slug não existe no modelo
4. `TelegramBotStatus.ATIVO` → `TelegramBotStatus.ATIVA`
5. `patch("asyncio.get_event_loop")` ineficaz → patch direto em `_executar_agente_telegram`
6. `import { api as baseApi }` não usado em audioClient.ts → quebrava vue-tsc no prod-build
