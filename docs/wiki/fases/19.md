## Objetivo

Persistir o histórico de conversas do chat no banco de dados, permitindo que usuários retomem conversas anteriores, visualizem o histórico e arquivem conversas antigas.

---

## O que foi implementado

### Backend — `src/docagent/conversa/`

```
conversa/
├── models.py     — Conversa, MensagemConversa
├── schemas.py    — ConversaPublic, MensagemConversaPublic, ConversaDetalhada, ConversaListResponse
├── services.py   — ConversaService (criar, listar, get, arquivar, restaurar)
└── router.py     — endpoints REST + lógica de paginação
```

**Modelo `Conversa`:**
- `agente_id` FK → agente
- `usuario_id` FK → usuario
- `titulo` (auto-gerado da primeira mensagem)
- `arquivada` bool
- Relationship → `mensagens: list[MensagemConversa]`

**Modelo `MensagemConversa`:**
- `conversa_id` FK → conversa
- `role`: `user | assistant | tool | system`
- `conteudo` Text
- `tool_name` (nullable, para mensagens de tool call)

### Endpoints

| Método | Path | Descrição |
|--------|------|-----------|
| GET | `/api/chat/conversas` | Lista conversas com paginação (`page`, `page_size`, `agente_id`, `arquivada`) |
| GET | `/api/chat/conversas/{id}` | Detalhe com todas as mensagens |
| DELETE | `/api/chat/conversas/{id}` | Arquiva (soft delete) |
| POST | `/api/chat/conversas/{id}/restaurar` | Restaura conversa arquivada |

**`POST /chat`** foi enriquecido:
- Aceita `conversa_id` opcional no body
- Se não fornecido, cria nova `Conversa` automaticamente
- Persiste cada mensagem user/assistant como `MensagemConversa`

### Frontend — `frontend/src/views/chat/`

- Sidebar esquerda: lista de conversas com agente, data, total de mensagens
- Paginação: botão "Carregar mais"
- Abas: "Ativas" e "Arquivadas"
- Ações: arquivar, restaurar
- Auto-seleção do primeiro agente ao montar `ChatView`
- Gravação de áudio no chat: botão de microfone, envio via `/api/stt`
- Reprodução de áudio nas respostas

### Alembic

## Gotchas

- `conversa_id` no body do `/chat` é opcional; se omitido, cria nova conversa silenciosamente.
- O título é gerado a partir das primeiras ~50 chars da mensagem inicial do usuário.
- `MensagemConversa.role` armazena como string (não enum) para compatibilidade com LangChain.
